Phase 2 — Minimal vertical slice Weeks 3–7 next

  • Week-3 spike validates the stack (gradient correctness vs finite differences, steps/sec); hard kill criterion → custom autograd fallback.
  • Single-agent RSSM in TS on a trivial discrete-action env. Known-good invariants ported as executable tests: KL free-bits floor, WM/AC gradient separation, λ-returns, NaN → graceful halt. Continue/termination head from day one.
  • JSONL telemetry + manifest.json per run — a run without a manifest didn't happen. No dashboard yet.
Gate G2 — loss decreases across 3 seeds; invariant tests green; the human has personally implemented ≥2 modules (replay buffer, λ-returns) with Claude reviewing.

Phase 3 — MARL envs + WM variants Weeks 8–12 next

  • One deterministic TS particle env (simple-spread-style, 2–3 agents, seeded RNG, pure functions, browser-renderable).
  • Exactly the 2–3 world-model variants the chosen question needs (e.g., independent per-agent WM · shared WM · opponent-latent WM) behind one interface.
Gate G3 — each variant trains NaN-free for 3 seeds; property-based env tests pass; the human implemented one variant's model head.

Phase 4 — Experiments → findings Weeks 13+ next

  • Reproduction-spec-first: grid, seeds (n ≥ 5), metrics, analysis plan co-signed before long runs. Mean ± std and IQM; baselines get equal tuning budget; figures from committed scripts + CSVs.
  • Artifacts: short-paper/blog writeup · interactive browser demo (trained WM client-side via WebGPU) · tagged v1.0 with reproduction instructions.
Gate G4 (Skeptic) — adversarial review in a fresh Claude session; BLOCKING/NOTE/NIT severities; failed claims land in Limitations, not silence.

The daily loop — Loop Engineering

Contract

loop/GOAL.md — standing mission + allow / approve / deny boundaries. Only the human edits it, at phase gates. One bounded increment per run; output is always a PR, never a push to main.

Stand-up (async)

Each run writes reports/standup/YYYY-MM-DD.md: Done · Learned · Blockers · Decisions needed · Tomorrow. The human replies in the PR; the next run reads replies first — that's the meeting.

Graduation levels

L1 research/docs only (now) → L2 tests + smoke training runs → L3 pre-signed experiment grids. Promotion = human decision after ≥3 clean runs. Safety valve: 2 unreviewed stand-ups → loop restricts itself to documentation.

Co-authorship protocol

MechanismRule
ADRsEvery irreversible decision is a ~15-line ADR merged via a PR the human approves.
Explain-before-implementCore algorithms get a 1-page explainer first; merge condition = the human summarizes it in their own words in LEARNING.md.
Role flipsuser-implements issues: human writes the code, Claude reviews. ≥2 per phase. Nobody verifies their own work.
Claimsself_checked may inform; only verified may justify decisions or enter the writeup. Every claim ties to an artifact path.
GatesClaude drafts the rubric; the human executes the review. A phase is done when the human says so.

Risks

RiskMitigation
TF.js maturity/correctnessWeek-3 spike with hard kill criterion; thin tensor-op boundary so a custom autograd swaps in without touching model code.
Laptop performance ceilingSmall dims (h≈256, z≈32), ≤~50k steps/run, overnight budget; steps/sec measured before committing to the grid.
Framework scope creepNo infrastructure the next experiment doesn't require; dashboard/demo deferred to Phase 4.
Question dead on arrivalKill criteria in every proposal; backup question pre-selected at G1; a clean negative result with ablations is still a finding.
Loop driftBounded increments; GOAL.md boundaries; safety valve after 2 unreviewed stand-ups; graduation levels gate what it may run.
Remote env limits (time-boxed, CPU)Checkpoint/resume experiments; commit only manifests + summary CSVs.
