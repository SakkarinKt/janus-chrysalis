# Contributing — the co-authorship protocol

This project is co-authored by a human (@SakkarinKt) and Claude. This file is the versioned contract for how that works. External contributors are welcome and follow the same rules — they exist so that *nobody verifies their own work*, human or model.

## Decisions

- **ADRs are the signoff surface.** Every irreversible decision (license, stack, research question, experiment spec) is a ~15-line record in `docs/adr/`, merged via a PR that the **human** approves. If it isn't in an ADR, it isn't decided.
- Numbered sequentially: `docs/adr/NNNN-short-title.md`. Status: `proposed` → `accepted` (or `superseded-by-NNNN`).

## Code and knowledge flow

- **Explain-before-implement.** Before implementing a core algorithm, Claude writes a 1-page note in `docs/explainers/`. The merge condition: the human writes a 3-sentence summary *in their own words* into `LEARNING.md`. If they can't, the explainer failed — iterate on the explainer, not the summary.
- **Role flips.** Issues labeled `user-implements` are coded by the human with Claude as PR reviewer — at least 2 per phase.
- **PRs even solo.** All non-trivial work lands via PR; the other author reviews. Direct-to-main is allowed only for typos and journal entries.
- **Conventional commits**: `feat:`, `fix:`, `exp:` (experiment runs/configs), `docs:`, `adr:`, `test:`, `chore:`.

## Claims and evidence

- Every empirical claim ties to an artifact path (`artifacts/<run>/…`). A run without a `manifest.json` didn't happen.
- Claims carry a verification status: `self_checked` may *inform* work; only `verified` (reviewed by the non-authoring party or an independent session) may *justify decisions* or enter the writeup.
- **Skeptic gate (G4):** before any claim lands in the writeup, an adversarial review in a fresh Claude session (no authoring context) applies the critique rubric with severities `BLOCKING` / `NOTE` / `NIT`. The human adjudicates disputes. Claims that fail land in Limitations, not in silence.
- Experiment protocol: seeds and n reported always; baselines get equal tuning budget; figures are generated by committed scripts from committed summary CSVs.

## Phase gates

Each phase ends with a gate (G1–G4) where **Claude drafts the rubric and the human executes the review**. A phase is done when the human says so, not when the code exists. Gates and phase definitions live in `PLAN.html`.

## The daily loop

A scheduled Claude agent runs once a day in the cloud under the contract in `loop/GOAL.md` (allow / approve / deny boundaries, graduation levels L1–L3). Each run opens a PR with a stand-up report in `reports/standup/`. Replying to "Decisions needed" items in that PR **is** the async stand-up meeting; the next run reads replies first. If two consecutive stand-ups go unreviewed, the loop restricts itself to research/documentation until the human catches up.

## Learning journal

`LEARNING.md` gets one short weekly entry from each author — including Claude (what assumptions broke, what the human taught it about intent). This is raw material for the final writeup, not ceremony.
