## Learned

- **[medium, self_checked]** Gating `update()` rather than `act()` on the `Policy` interface keeps
  the freeze predicate (`isFrozen`) fully decoupled from the rollout loop — a future metrics module
  can reproduce the same per-step frozen/training classification independently from
  `EpisodeStepRecord.frozen` without re-deriving it from rollout internals. This matters because
  proposal `0001`'s drift-attributable metric needs exactly this classification to know which
  post-freeze transitions belong to which condition.
- **[low, self_checked]** Because `RandomPolicy` has no `update`, this increment's tests can only
  confirm the *wiring* is correct (via a counting test double), not that freezing has any
  observable effect on behavior yet — that only becomes testable once a real learning policy or
  the world-model cell exists. Flagging so this isn't mistaken later for having validated anything
  about drift-attributable error itself.
