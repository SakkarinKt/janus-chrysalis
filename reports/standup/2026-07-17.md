## Learned

- **[high, self_checked]** `tf.layers.gruCell` cannot be driven directly via bare
  `Layer.apply([input, state])` — its `build()` method rejects a two-shape input array. The
  documented, working pattern for step-by-step control over a cell's initial state is
  `tf.layers.rnn({cell, returnState: true}).apply(sequenceTensor, {initialState: [state]})` on a
  single-timestep sequence. This is a concrete API detail the literature/robustness-note research
  couldn't have surfaced (it's an implementation fact, not a design question) — confirmed by
  actually running it against the real dependency in this session rather than assuming from
  TF.js's docs.
- **[medium, self_checked]** The reviewer's "for the cell PR" framing for the ADR-0002 §7/§6
  wording cleanup reads correctly as "whichever PR starts the cell," not "the PR that writes the
  STE code specifically" — today's PR is the first cell-code PR, so this is the natural point to
  land it, even though the STE code the phrase describes is still unwritten. Flagged as an
  assumption below in case a stricter reading was intended.
