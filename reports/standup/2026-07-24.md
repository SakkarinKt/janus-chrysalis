## Learned

- **[high, self_checked, execution-confirmed]** The 2026-07-23 replay-ratio "diagnosis" (this run's
  main fix) was itself backwards, not just imprecise — a second-order correction on top of a
  same-day first correction. Worth carrying forward explicitly: when correcting a unit-conversion
  bug, re-derive the "what did the old number actually equal" arithmetic rather than asserting an
  intuitive-sounding assumption (here, "must have assumed ratio 1" felt right but the actual math
  said ratio 16) — this is the second time in this project's history a same-day self-correction
  needed correcting (see PR #20's `tf.stopGradient`→`tf.customGrad` reversal for the first).
- **[high, self_checked, execution-confirmed]** `notes/rssm.test.ts`'s finite-difference chained-
  gradient test (test 44, "chaining step()+prior() across >=2 timesteps... matches finite
  differences") is flaky: failed 2 of 4 consecutive `npm test` runs today with no code changes
  between runs, no `src/`/`test/` diff in this increment, and no stack trace beyond the assertion
  itself in the failure output. Not investigated further today (out of scope for a PR-feedback
  processing increment; discovering it wasn't the goal), but it's a new, previously-unrecorded data
  point about the finite-difference gradient-check's numerical stability — worth root-causing (tighter
  tolerance vs. genuine numerical instability in the chained STE path) before it's trusted as a gate
  on future RSSM changes.
- **[medium, self_checked, execution-confirmed]** Backfilling historical run data from squash-merged
  PR commits works via the GitHub API (`get_commit`/`get_file_contents` against a commit SHA) even
  after the source branch is deleted — GitHub keeps unreferenced commits reachable for a period
  after a squash merge. Useful precedent: don't assume intermediate commits inside a squash-merged PR
  are permanently unrecoverable.
