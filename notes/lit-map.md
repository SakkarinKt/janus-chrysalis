## Design axis this map is organized around

The mission statement's central question is where a world model sits relative to the agents that co-learn in the same environment:

1. **Fully shared** — one world model, joint state/observations, all agents' policies condition on it.
2. **Per-agent + centralized aggregation** — separate world model per agent, but a mechanism (attention, mixing, shared upper-level latent) explicitly pools information to counter non-stationarity. → **MARIE**
3. **Per-agent + bi-level (train-time-only sharing)** — global info baked into an upper-level latent used only during training; execution uses a lower level that's already been informed by it, so no runtime sharing is needed at all. → **MABL**
4. **Per-agent + explicit peer-to-peer communication** — no central aggregator; agents exchange messages (GNN-style) that shape each other's world-model state directly. → **CoDreamer**
5. **Per-agent + comms via imagined rollouts, PPO-style policy update** — the original model-based-MARL reference point everything else is measured against. → **MAMBA**
6. *(Not yet mapped)* **Fully independent, no sharing at all** — the non-stationarity stress case / naive baseline every other paper implicitly argues against. Need to find or construct this as an ablation, not a citation — it's usually the strawman other papers compare to, rarely a paper in its own right.

## Preliminary gap observations (self_checked, low–medium confidence — 8 papers, growing evidence but not yet ≥10; re-check again once the map is complete)

- **[low, verified]** All 7 *empirical* papers evaluate primarily on final-return / sample-efficiency curves — none report a *direct, isolated* measurement of non-stationarity itself (e.g., world-model prediction error over training as a co-learning partner's policy shifts, holding everything else fixed). (The 8th, "Dreaming of Others," is a proposal paper with no experiments at all, so it's excluded from this observation rather than counted as a data point either way.) Holds up across all 7 empirical papers — strengthening (not just preliminary) candidate gap: existing work treats non-stationarity as a problem to architect *around*, not a quantity to *measure*.
- **[low, self_checked]** No paper in the map runs the same architecture across a shared-vs-per-agent sweep on one fixed environment/task — each paper introduces one new architecture and compares it to prior papers' reported baselines (different codebases, possibly different hyperparameter budgets). A controlled, single-codebase sweep across the axis below, on one laptop-scale env, would be a novel (if modest) empirical contribution distinct from proposing a 6th (or 7th) architecture.
- **[medium, self_checked]** Non-stationarity mitigation now has *three* structurally distinct mechanisms in the map, not just "architecture choice": (1) architecture-level aggregation (MARIE, GAWM), (2) bi-level train-time-only latent sharing (MABL), (3) training-procedure-level recency-weighted replay (MATWM), independent of architecture. This suggests the controlled sweep's design space is at least two-dimensional — sharing *topology* × non-stationarity-mitigation *mechanism* — rather than one axis of competing architectures. MATWM's replay mechanism in particular looks factorizable: it could plausibly be layered onto any of the other architectures as an independent variable, which would make for a cheap, high-signal ablation.
- **[medium, verified]** MMSA's boundary-case question is now resolved by primary-source read: its world model **is** joint/shared (not per-agent), trained centrally on aggregated joint state-action embeddings, with CTDE-style decentralized execution and a value-decomposition mixing network layered on top. So MMSA isn't cleanly "orthogonal" to the sharing-topology axis after all — it's closest to axis point 1 ("fully shared"), but point 1's current wording ("all agents' policies condition on it") doesn't quite fit since MMSA's policies act on local observations at execution time. Suggests the axis may need a separate policy-execution-scope dimension alongside world-model-sharing scope — worth resolving explicitly in the next lit-map increment rather than leaving point 1 unfilled.
- **[low, self_checked]** Of the three papers added this session, only MATWM and GAWM name non-stationarity (of different flavors — agent-policy non-stationarity for MATWM, world-model-training instability for GAWM) as a first-class target; MMSA's stated motivation (data efficiency via richer joint embeddings) doesn't mention it. Combined with MAMBA/MABL/CoDreamer from the first batch, roughly half the map so far treats non-stationarity as a named target and half treats it as an implicit consequence of sharing design — worth quantifying precisely once the map hits 10.

## Non-stationarity / opponent-modeling-in-latent-space seed area

Growing: MARIE (architecture-level aggregation), GAWM (world-model-training-stability framing), MATWM (training-procedure-level replay prioritization), and now "Dreaming of Others" (latent teammate modeling / Theory-of-Mind) give four structurally distinct mechanisms — though the last is a proposal paper with no empirical results, so it's a design-input reference rather than a baseline. Still this map's thinnest-evidenced area *empirically* relative to the sharing-topology axis — continue prioritizing it for the next increment, especially citation-chasing from MATWM's related work (it explicitly compares itself against MARIE and likely cites other non-stationarity-focused MARL papers), ideally surfacing an empirical (not just conceptual) addition next.
