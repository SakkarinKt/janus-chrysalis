## Design axis this map is organized around

The mission statement's central question is where a world model sits relative to the agents that co-learn in the same environment:

1. **Fully shared** — one world model, joint state/observations, all agents' policies condition on it.
2. **Per-agent + centralized aggregation** — separate world model per agent, but a mechanism (attention, mixing, shared upper-level latent) explicitly pools information to counter non-stationarity. → **MARIE**
3. **Per-agent + bi-level (train-time-only sharing)** — global info baked into an upper-level latent used only during training; execution uses a lower level that's already been informed by it, so no runtime sharing is needed at all. → **MABL**
4. **Per-agent + explicit peer-to-peer communication** — no central aggregator; agents exchange messages that shape each other's world-model state or decisions directly. → **CoDreamer** (raw latent state via GNN); **Communicating Plans, Not Percepts** shows this point has its own sub-dimension — *what* gets communicated (raw state vs. a compressed self-intention/plan vs., per Dreaming of Others, an inferred prediction *about* a teammate with no communication at all) is a separable design choice from the topology itself.
5. **Per-agent + comms via imagined rollouts, PPO-style policy update** — the original model-based-MARL reference point everything else is measured against. → **MAMBA**
6. *(Not yet mapped)* **Fully independent, no sharing at all** — the non-stationarity stress case / naive baseline every other paper implicitly argues against. Need to find or construct this as an ablation, not a citation — it's usually the strawman other papers compare to, rarely a paper in its own right.

## Preliminary gap observations (self_checked, low–medium confidence — 10 papers, target reached; ready for gap-analysis drafting, though most items below still want a primary-source pass before being load-bearing for a proposal)

- **[low, verified]** All 8 *empirical, multi-agent* papers evaluate primarily on final-return / sample-efficiency curves — none report a *direct, isolated* measurement of non-stationarity itself (e.g., world-model prediction error over training as a co-learning partner's policy shifts, holding everything else fixed). ("Dreaming of Others" is a proposal paper with no experiments, and Drama is single-agent, so both are excluded from this observation rather than counted as data points either way.) Holds up across all 8 empirical multi-agent papers — strengthening (not just preliminary) candidate gap: existing work treats non-stationarity as a problem to architect *around*, not a quantity to *measure*.
- **[low, self_checked]** No paper in the map runs the same architecture across a shared-vs-per-agent sweep on one fixed environment/task — each paper introduces one new architecture and compares it to prior papers' reported baselines (different codebases, possibly different hyperparameter budgets). A controlled, single-codebase sweep across the axis below, on one laptop-scale env, would be a novel (if modest) empirical contribution distinct from proposing a 6th (or 7th) architecture.
- **[medium, self_checked]** Non-stationarity mitigation now has *three* structurally distinct mechanisms in the map, not just "architecture choice": (1) architecture-level aggregation (MARIE, GAWM), (2) bi-level train-time-only latent sharing (MABL), (3) training-procedure-level recency-weighted replay (MATWM), independent of architecture. This suggests the controlled sweep's design space is at least two-dimensional — sharing *topology* × non-stationarity-mitigation *mechanism* — rather than one axis of competing architectures. MATWM's replay mechanism in particular looks factorizable: it could plausibly be layered onto any of the other architectures as an independent variable, which would make for a cheap, high-signal ablation. (Drama's DFS is a superficially similar replay-reweighting idea but targets a different failure mode — early-training world-model inaccuracy, not co-learning non-stationarity — so it's a separate ablation, not a second data point for this mechanism.)
- **[medium, verified]** MMSA's boundary-case question is now resolved by primary-source read: its world model **is** joint/shared (not per-agent), trained centrally on aggregated joint state-action embeddings, with CTDE-style decentralized execution and a value-decomposition mixing network layered on top. So MMSA isn't cleanly "orthogonal" to the sharing-topology axis after all — it's closest to axis point 1 ("fully shared"), but point 1's current wording ("all agents' policies condition on it") doesn't quite fit since MMSA's policies act on local observations at execution time. Suggests the axis may need a separate policy-execution-scope dimension alongside world-model-sharing scope — worth resolving explicitly in the next lit-map increment rather than leaving point 1 unfilled. (A strong fully-shared-topology candidate, "From Pixels to Cooperation," arXiv 2511.01310, was found this session and briefly considered for exactly this gap — but it was withdrawn by its author and is excluded; see run note above. Point 1 remains unfilled by a clean example.)
- **[low, self_checked]** Of the three papers added 2026-07-05, only MATWM and GAWM name non-stationarity (of different flavors — agent-policy non-stationarity for MATWM, world-model-training instability for GAWM) as a first-class target; MMSA's stated motivation (data efficiency via richer joint embeddings) doesn't mention it. Combined with MAMBA/MABL/CoDreamer from the first batch, roughly half the *multi-agent* map treats non-stationarity as a named target and half treats it as an implicit consequence of sharing design — now quantifiable at n=8 empirical multi-agent papers rather than a preliminary read.
- **[medium, self_checked]** The axis-4 (explicit communication) entries now differ in *what* is communicated, not just topology: CoDreamer shares raw latent state every step; "Communicating Plans, Not Percepts" shares a compressed self-intention/plan generated by imagining one's own rollout. Neither shares a model *of* a teammate — that's "Dreaming of Others"' distinct ToM-inference approach, which needs no communication channel at all. Suggests the gap-analysis draft should treat "communication content" and "sharing topology" as two separate axes rather than folding content into the topology axis as an afterthought.

## Non-stationarity / opponent-modeling-in-latent-space seed area

Growing: MARIE (architecture-level aggregation), GAWM (world-model-training-stability framing), MATWM (training-procedure-level replay prioritization), and now "Dreaming of Others" (latent teammate modeling / Theory-of-Mind) give four structurally distinct mechanisms — though the last is a proposal paper with no empirical results, so it's a design-input reference rather than a baseline. Still this map's thinnest-evidenced area *empirically* relative to the sharing-topology axis — continue prioritizing it for the next increment, especially citation-chasing from MATWM's related work (it explicitly compares itself against MARIE and likely cites other non-stationarity-focused MARL papers), ideally surfacing an empirical (not just conceptual) addition next.
